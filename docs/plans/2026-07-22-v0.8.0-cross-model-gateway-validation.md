# tldr-swinton 0.8.0 and Cross-Model Gateway Validation Plan

> **For Codex:** Execute this plan sequentially in the current session. The release must precede the external validation campaign so every evaluation cell can record the same frozen `tldrs` version.

**Goal:** Repair the macOS release validator, freeze and publish the Context Gateway as `tldr-swinton` 0.8.0, then measure its coding correctness and end-to-end token savings on pinned external repositories using two languages and two model harnesses.

**Architecture:** Keep release validation as a thin portable wrapper around Sylveste's shared checker, with an explicit override for nonstandard installations and deterministic sibling/monorepo discovery. Generalize the existing paired agent evaluator around a runner protocol and a pinned source-repository input while preserving its hidden mutation/grader boundary. Every baseline/adaptive pair materializes from the same recorded Git SHA; packet/runtime and packet-budget variants are promoted only when hidden-grader correctness is non-inferior.

**Tech Stack:** Bash, Python 3.11+, pytest, Git, `ic publish`, Codex CLI JSONL, Claude Code headless `stream-json`, YAML task corpora, Markdown/JSON reports.

## Must-Haves

**Truths**
- `scripts/check-versions.sh` works from this standalone macOS checkout and from the historical Sylveste monorepo layout.
- Version files and marketplace metadata agree on `0.8.0`, the live launcher reports `0.8.0`, and the release commit/tag is pushed.
- Each evaluation run records the evaluated repository URL/path, pinned source SHA, task-corpus hash, harness name/version, model, policy, and packet budget.
- Baseline and adaptive cells for a task use identical source bytes and mutation bytes; only the treatment guidance/tool exposure differs.
- The external campaign covers at least two repositories, two implementation languages, and both Codex and Claude Code.
- A variant is rejected if hidden-grader coding correctness regresses, regardless of token savings.

**Artifacts**
- A regression test for portable shared-version-checker discovery.
- A runner-neutral agent-evaluation API plus tested Codex and Claude Code adapters.
- External repository manifests and hidden mutation/grader assets that are not copied into agent workspaces.
- Machine-readable outcomes plus a dated same-SHA validation report with correctness, token, latency, and contamination gates.

**Key Links**
- `scripts/check-versions.sh` resolves the shared Sylveste checker and is exercised by `tests/test_install_script.py`.
- `tldr-bench/tldr_bench/agent_eval/cli.py` selects a source manifest and runner, while `workspace.py` archives the pinned source checkout.
- `codex_runner.py` and the new Claude runner normalize both harnesses into `TraceMetrics` and a common process-result contract.
- External task YAML points to evaluator-owned mutation and grader files; source manifests pin repositories to immutable SHAs.
